# 計画の作成 (Plan Creation)

./plans ディレクトリにマークダウンファイルを作成した際は、必ず `codex-review` スキルを使用してCodexによるレビューを受けてください。

# 実行計画 (ExecPlans)

複雑な機能や大規模なリファクタリングを行う場合は、設計から実装まで（.agent/PLANS.md に記載されているように）ExecPlanを使用してください。

# レビューゲート (Review gate / codex-review)

主要なマイルストーン（仕様/計画の更新後、主要な実装段階（5ファイル以上 / パブリックAPI / インフラ設定）、およびコミット/PR/リリースの前）では、`codex-review` スキルを実行し、レビュー→修正→再レビューのサイクルをクリーンになるまで繰り返してください。

# タスク管理 (Task Management)

機能の実装やコードの変更を行う際は、Tasks機能を使用して進捗を管理・追跡してください。作業を明確なステップに分解し、進行に合わせてタスクのステータスを更新してください。

# その他 (Other)

決定を求める際は、"AskUserQuestion" を使用してください。
