# ブログ記事ルール

## 文体・トーン

- 読者に語りかけるような親しみやすい文体を使うこと
- 「です・ます」調で統一すること
- 専門用語を使う場合は初出時に簡単な説明を添えること
- 一文は60文字以内を目安に、読みやすさを優先すること

## 記事構成

- タイトルは具体的で、読者が得られるメリットが分かるようにすること
- 導入文（リード）は3行以内で、記事を読む理由を明示すること
- H2見出しは5〜8個を目安に、論理的な流れで構成すること
- 各セクションに具体例やコード例を含めること
- まとめセクションでは要点を箇条書きで整理すること

## SEO 基本チェック

- タイトルに主要キーワードを含めること（60文字以内）
- メタディスクリプションを150文字以内で記載すること
- H2/H3見出しに自然な形でキーワードを含めること
- 画像には必ず alt テキストを設定すること
- 内部リンク・外部リンクを適切に配置すること

## 日本語表記統一

- 半角英数字の前後に半角スペースを入れること（例: これは JavaScript の話です）
- 数字は半角を使用すること
- 括弧は全角「（）」を基本とし、コード内では半角「()」を使用すること
- 「！」「？」の後には全角スペースを入れること

## 画像・メディア

- アイキャッチ画像は必ず設定すること
- 記事内の画像は WebP 形式を推奨
- スクリーンショットには手順番号や矢印で補足すること
